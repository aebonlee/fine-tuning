## 3. 접근성 (4/10, +1) — CRITICAL

### 1차 대비 개선사항
- Toast에 `role="status"`, `aria-live="polite"` 추가 (`ToastContext.jsx:30`)

- 로딩 스피너에 `role="status"` 추가 (`PublicLayout.jsx:27`)
- Skip-to-content 링크 CSS 구현 (`base.css:284-299`)
- BoardDetail 모달에 `role="dialog"`, `aria-modal="true"` 추가
- Navbar 모바일 메뉴에 `aria-expanded`, `aria-controls` 추가

### 여전히 부족한 사항

#### WCAG A 위반 (즉시 수정 필요)
- **focus-visible 상태 전무**: 전체 CSS에서 `:focus-visible` 선언 40곳 이상 누락
  - `.nav-link`, `.btn-link`, `.carousel-arrow`, `.carousel-dot`
  - `.quick-access-card`, `.board-card`, `.category-lesson-item`
  - `.page-btn`, `.settings-nav-item`, `.ck-nav-parent/child`
- **키보드 내비게이션 부족**: 캐러셀 화살표/점, 사이드바 아코디언 키보드 조작 불가
- **모달 포커스 트랩 미구현**: BoardDetail 삭제 모달에서 Tab 키로 모달 밖 이동 가능
- **일부 form label 누락**: 일부 입력 필드에 placeholder만 사용

#### WCAG AA 위반
- **색상 대비**: 푸터 링크 `rgba(255,255,255,0.5)` 대비 약 3.5:1 → 기준 4.5:1 미달
- **터치 타겟 부족**: 여러 요소가 44px 미만 (상세는 반응형 섹션 참조)
