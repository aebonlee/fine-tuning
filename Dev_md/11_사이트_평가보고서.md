## 1. 코드 품질 & 아키텍처 (7/10)

### 강점
- 깔끔한 컴포넌트 분리: pages, layouts, components, contexts 디렉토리 구조
- 모든 페이지 `React.lazy()` + `Suspense` 코드 분할
- Context 기반 상태 관리 (Theme, Language, Auth, Toast)
- 재사용 가능한 SEO 컴포넌트

### 개선 필요
- **중복 상수**: `levelColors`가 Home.jsx, LessonCategories.jsx, LessonList.jsx에 중복 정의
- **중복 상수**: `CATEGORY_CLASSES`가 Board.jsx, BoardDetail.jsx에 중복 정의
- **번역 방식 혼재**: `t()` 함수와 `language === 'ko' ? ... : ...` 인라인 분기가 60+ 곳에서 혼용
- **인라인 스타일 산재**: Settings.jsx, LessonList.jsx, BoardWrite.jsx 등에서 `style={{...}}` 사용
- **React Error Boundary 없음**: 런타임 에러 발생 시 전체 앱 크래시

## 2. SEO (8/10)

### 강점
- index.html에 OG/Twitter 메타 태그 완비
- 페이지별 SEO 컴포넌트로 동적 메타 태그 관리
- Canonical URL 자동 생성
- JSON-LD 구조화 데이터 (Home: WebSite, LessonDetail: BreadcrumbList)

### 개선 필요
- `<html lang="ko">` 고정 → 언어 전환 시 반영 안 됨
- `hreflang` 태그 없음 (한/영 교차 참조)
- sitemap.xml, robots.txt 없음
- 대부분의 페이지에 JSON-LD 구조화 데이터 미적용

## 3. 접근성 (3/10) - CRITICAL

### 주요 문제
- **ARIA 속성 전무**: 전체 코드베이스에 `aria-label`, `aria-expanded`, `role` 등 없음
- **키보드 내비게이션 미지원**: 드롭다운, 모달에 `onKeyDown` 핸들러 없음
- **Skip-to-content 링크 없음**: 키보드 사용자가 네비게이션을 건너뛸 수 없음
- **Form label 미연결**: Login/Register 입력 필드에 `<label>` 없이 placeholder만 사용
- **모달 포커스 트랩 없음**: 삭제 확인 모달에 `role="dialog"`, `aria-modal` 없음
- **로딩 상태 접근성 없음**: 스피너에 `aria-busy`, `role="status"` 없음
- **prefers-reduced-motion 미지원**: 애니메이션 비활성화 불가

## 4. 성능 (7/10)

### 강점
- 코드 분할 (lazy loading 모든 페이지)
- 레슨 콘텐츠 동적 import
- 검색 디바운스 (400ms)
- useMemo로 비용 높은 계산 캐싱
- Google Fonts preconnect

### 개선 필요
- **Font Awesome 전체 로드** (~90KB): 사용하는 아이콘만 트리셰이킹 필요
- **Google Fonts 6종 웨이트**: 실제 사용하는 4종만 로드 권장
- **BoardDetail 4개 순차 쿼리**: prev/next + comments는 Promise.all()로 병렬화 가능
- **히어로 파티클 애니메이션**: 모바일에서도 계속 실행 → GPU 부하
- **이미지 lazy loading 없음**: `<img>` 태그에 `loading="lazy"` 미적용
- **backdrop-filter: blur()**: 저사양 모바일 기기에서 성능 부하

## 5. 모바일 반응형 (6/10)

### 강점
- 4단계 브레이크포인트: 1200px, 1024px, 768px, 480px
- 모바일 전용 네비게이션 메뉴 (아코디언 서브메뉴)
- 모바일 메뉴 오픈 시 body scroll lock
- 레슨 사이드바 토글 버튼

### 주요 문제 (모바일 최적화 대상)

#### 터치 타겟 위반 (< 44px)
| 요소 | 파일 | 현재 크기 | 심각도 |
|------|------|-----------|--------|
| `.btn-sm` | base.css | ~30px | HIGH |
| `.mobile-color-dot` | navbar.css | 28px | HIGH |
| `.toast-close` | toast.css | ~20px | HIGH |
| `.comment-delete-btn` | community.css | ~23px | HIGH |
| `.page-btn` (페이지네이션) | community.css | 36px→32px@480 | HIGH |
| `.ck-nav-child` (사이드바) | intro-page.css | ~30px | HIGH |
| `.color-picker-btn` | navbar.css | 36px | MEDIUM |
| `.theme-toggle` | navbar.css | 36px | MEDIUM |
| `.user-avatar-btn` | navbar.css | 36px | MEDIUM |
| `.mobile-ctrl-btn` | navbar.css | 40px | LOW |

#### 누락된 반응형 규칙
- **1024px**: Dashboard grid, Settings layout, 커뮤니티 게시판 필터 미적용
- **480px**: Toast 컨테이너 (max-width: 380px 오버플로우), Settings 패딩/폰트 미적용
- 캐러셀 스와이프 제스처 없음 (화살표 1024px에서 숨김)
- 마크다운 테이블 가로 스크롤 불가 (overflow-x: hidden으로 잘림)
- 히어로 파티클 애니메이션 모바일에서 미숨김

#### 폰트 & 간격
- 푸터 링크 대비 비율 3.5:1 → WCAG AA 기준(4.5:1) 미달
- `.lessons-page` 상단 패딩 100px 하드코딩 → `var(--nav-height)` 참조 필요
- 네비바 내부 패딩 40px→16px 점프 (1024px 단계 없음)

## 6. UI/UX 디자인 (6/10)

### 개선 필요
- **로딩 표시**: "..." 만 표시 → 스피너 또는 "Loading..." 텍스트 필요
- **대시보드 학습 진도**: 항상 `0/N`으로 표시 → 미구현 기능
- **게시글 내용**: 리치 텍스트/마크다운 에디터 없음 (plain textarea)
- **삭제 확인**: 게시글 제목이나 댓글 내용 미리보기 없음
- **비저장 경고**: 글쓰기 페이지에서 뒤로가기 시 확인 없음
- **검색 범위**: 제목만 검색 → 내용 검색 미지원
- **"Post not found"**: 영어 하드코딩 → 다국어 미적용
- **Back to Top 버튼**: 긴 레슨 페이지에 없음

## 7. 보안 (5/10)

### 주요 문제
- **Admin 이메일 클라이언트 노출**: `admin.js`에 관리자 이메일 하드코딩 → 번들에서 확인 가능
- **Supabase 플레이스홀더 폴백**: 환경변수 누락 시 'placeholder' 값으로 자동 대체 → 무음 실패
- **쿠키 보안 속성 없음**: `SameSite`, `Secure` 플래그 미설정
- **입력 검증 부재**: 회원가입 이름 필드에 길이/문자 제한 없음
- **게시글 수정 시 소유자 검증**: RLS에 의존하지만 클라이언트 쿼리에 `user_id` 필터 없음
- **조회수 레이스 컨디션**: RPC 실패 시 폴백 수동 증가에서 동시성 문제 가능

## 8. 다국어 (5/10)

### 주요 문제
- **Register.jsx 영어 하드코딩**: 비밀번호 불일치, 이메일 확인 메시지 등 영어 고정
- **번역 키 누락**: 폼 검증 메시지, 에러 상태, 검색 필터 UI
- **번역 방식 혼재**: `t()` 함수 vs 인라인 삼항 연산자 60+ 곳
- **날짜 포맷 로케일 미적용**: `toLocaleDateString()` 브라우저 기본 로케일 사용
- **저작권 연도 하드코딩**: `© 2025` 고정
- **index.html 한국어 전용**: `<title>`, `<meta>` 태그 한국어만

## 9. 다크모드 (7/10)

### 누락된 다크모드 오버라이드
- **마크다운 인라인 코드**: `rgba(27, 58, 107, 0.06)` → 다크 배경에서 불가시
- **마크다운 blockquote**: `rgba(27, 58, 107, 0.03)` → 다크 배경에서 불가시
- **카카오 소셜 버튼**: `#FEE500` 밝은 노란색 → 다크 배경에서 눈부심
- **인트로 페이지 카드**: 보더 색상이 미묘하게 표시
- **Toast 닫기 버튼 호버**: 다크모드 전용 오버라이드 없음
- **rgba(27,58,107,...) 하드코딩 15곳**: CSS 변수로 추출 필요
