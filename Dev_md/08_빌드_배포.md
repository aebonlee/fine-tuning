# FineTuning - 빌드 & 배포

## 개발 서버
```bash
npm run dev          # http://localhost:5176
```

## 프로덕션 빌드
```bash
npm run build        # dist/ 폴더에 빌드 결과물 생성
npm run preview      # 빌드 결과물 로컬 미리보기
```

## Vite 설정 (`vite.config.js`)
- **포트**: 5176
- **Manual Chunks**: vendor, supabase, markdown 분리
- **Base**: `/fine-tuning/` (GitHub Pages 배포 시)

## GitHub Pages 배포

### 방법: gh-pages 패키지 사용
```bash
npm install --save-dev gh-pages
```

### package.json scripts
```json
{
  "scripts": {
    "deploy": "npm run build && gh-pages -d dist"
  }
}
```

### 배포 명령
```bash
npm run deploy
```

### SPA 라우팅 처리
GitHub Pages는 SPA 라우팅을 지원하지 않으므로 `404.html`을 추가:
- `public/404.html`: `index.html`과 동일한 내용으로 리다이렉트 처리

## OG 이미지 생성
```bash
npm run generate-og  # scripts/generate-og-images.mjs 실행
```
- sharp 라이브러리로 SVG → PNG 변환
- `public/og/` 폴더에 11개 이미지 생성:
  - default.png, home.png, lessons.png, community.png
  - fundamentals.png, data.png, techniques.png
  - tools.png, models.png, production.png, projects.png
